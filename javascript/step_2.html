Express

In this example we will be using Express, the most popular web framework for Node.js, but you can use any other similar alternative. We chose Express because it comes with a myriad of HTTP utility methods and middleware, making it super quick and easy to use.

We start by requiring and initialising express at the top. Additionally, we set up a property port where we will store the port where your bot will be listening. Note how we set this in a way that we can use an environment variable to override the default port 8090. Finally, we use body-parser middleware to parse incoming requests as JSON. The "extended" syntax allows for rich objects and arrays to be encoded into the URL-encoded format, allowing for a JSON-like experience with URL-encoded. More information about body-parser can be found here.
